Architecture

Three layers, each feeding the next. Source data never gets modified.

LayerDatasetPurposeTables
Sourcecustomer_spendRaw FNB transaction data (read-only)transaction_data (2.2B rows), base_data (82M rows)
Staging stagingstagingCleaned, typed, deduplicatedstg_transactions, stg_customers
Analytics analyticsanalyticsFeature engineering, intermediate aggregationsint_customer_category_spend, int_rfm_features
Marts martsmartsBusiness-ready tables for dashboards and reports15+ tables (see below)

The pipeline runs via bash scripts/run.sh sandbox [step] where steps 0-6 build each layer sequentially. All SQL is in sql/ organised by layer.

Staging layer

stg_transactions staging

One row per card swipe. 2.24 billion rows. Partitioned by month for cost control.

ColumnTypeDescription
UNIQUE_IDSTRINGAnonymised customer identifier
EFF_DATEDATETransaction date
trns_amtFLOAT64Transaction amount (Rands)
DESTINATIONSTRINGMerchant name (e.g. WOOLWORTHS, CLICKS)

Cost warning: Any query that scans stg_transactions without a date filter costs ~R350. Always filter with WHERE EFF_DATE >= DATE_SUB((...), INTERVAL 12 MONTH).
-- Safe query: recent 12 months, grouped
SELECT DESTINATION, ROUND(SUM(trns_amt), 0) AS spend, COUNT(*) AS txns
FROM `fmn-sandbox.staging.stg_transactions`
WHERE EFF_DATE >= DATE_SUB(
    (SELECT MAX(EFF_DATE) FROM `fmn-sandbox.staging.stg_transactions`),
    INTERVAL 12 MONTH)
GROUP BY 1
ORDER BY spend DESC
LIMIT 20

Analytics layer

int_customer_category_spend analytics

One row per customer × destination. Pre-aggregated spend. This is the workhorse — most mart tables read from here instead of scanning raw transactions.

ColumnTypeDescription
UNIQUE_IDSTRINGCustomer
DESTINATIONSTRINGMerchant
CATEGORY_TWOSTRINGMerchant category
dest_spendFLOAT64Total spend at this destination

This table is cheap to query (~212M rows, but that's 30x smaller than raw transactions). Use it instead of stg_transactions whenever possible.
-- Who shops at Clicks, ranked by spend?
SELECT UNIQUE_ID, dest_spend, share_of_wallet_pct
FROM `fmn-sandbox.analytics.int_customer_category_spend`
WHERE DESTINATION = 'CLICKS'
ORDER BY dest_spend DESC

mart_cluster_output

One row per customer. Their ML-assigned behavioural segment.

ColumnDescription
UNIQUE_IDCustomer
segment_nameChampions, Loyal High Value, Steady Mid-Tier, At Risk, Dormant
centroid_idRaw cluster number from k-means
-- Segment sizes and spend
SELECT co.segment_name,
    COUNT(*) AS customers,
    ROUND(AVG(r.total_spend), 0) AS avg_spend,
    ROUND(AVG(r.recency_days), 0) AS avg_recency,
    ROUND(AVG(r.distinct_merchants), 0) AS avg_merchants
FROM `fmn-sandbox.marts.mart_cluster_output` co
JOIN `fmn-sandbox.analytics.int_rfm_features` r ON co.UNIQUE_ID = r.UNIQUE_ID
GROUP BY 1
ORDER BY avg_spend DESC

mart_customer_clv

One row per customer. Predicted future value from the linear regression model.

ColumnDescription
UNIQUE_IDCustomer
predicted_clvPredicted 6-month future spend
clv_tierPlatinum, Gold, Silver, Bronze, Basic
historical_spend12-month actual spend
-- Cross-reference: high-value customers at churn risk
SELECT co.segment_name, cr.churn_risk_level, clv.clv_tier,
    COUNT(*) AS customers,
    ROUND(SUM(clv.predicted_clv) / 1e6, 1) AS predicted_value_m
FROM `fmn-sandbox.marts.mart_cluster_output` co
JOIN `fmn-sandbox.marts.mart_churn_risk` cr ON co.UNIQUE_ID = cr.UNIQUE_ID
JOIN `fmn-sandbox.marts.mart_customer_clv` clv ON co.UNIQUE_ID = clv.UNIQUE_ID
WHERE cr.churn_risk_level IN ('Critical', 'High')
    AND clv.clv_tier IN ('Platinum', 'Gold')
GROUP BY 1, 2, 3
ORDER BY predicted_value_m DESC
This is the "red alert" query. Platinum CLV + Critical churn = your most valuable customers about to leave. Action these first.

mart_audience_catalog + mart_audience_members

The audience marketplace. Catalog has one row per audience (metadata). Members maps customers to audiences.

Column (catalog)Description
audience_idD01, L01, B01, etc.
audience_nameHuman-readable name
audience_typeDemographic, Lifestyle, Behavioral, Seasonal, Geographic, Cross-category

-- Which audiences overlap most?
SELECT a1.audience_name AS aud_1, a2.audience_name AS aud_2,
    COUNT(*) AS overlap,
    ROUND(COUNT(*) * 100.0 / LEAST(a1c.cnt, a2c.cnt), 1) AS overlap_pct
FROM `fmn-sandbox.marts.mart_audience_members` m1
JOIN `fmn-sandbox.marts.mart_audience_members` m2
    ON m1.UNIQUE_ID = m2.UNIQUE_ID AND m1.audience_id < m2.audience_id
JOIN `fmn-sandbox.marts.mart_audience_catalog` a1 ON m1.audience_id = a1.audience_id
JOIN `fmn-sandbox.marts.mart_audience_catalog` a2 ON m2.audience_id = a2.audience_id
JOIN (SELECT audience_id, COUNT(*) AS cnt FROM `fmn-sandbox.marts.mart_audience_members` GROUP BY 1) a1c ON a1c.audience_id = m1.audience_id
JOIN (SELECT audience_id, COUNT(*) AS cnt FROM `fmn-sandbox.marts.mart_audience_members` GROUP BY 1) a2c ON a2c.audience_id = m2.audience_id
GROUP BY 1, 2, a1c.cnt, a2c.cnt
ORDER BY overlap DESC
LIMIT 20

ML models

Three models, all built using BigQuery ML. The SQL that creates them is in sql/03_ml/.

K-means segmentation

-- The model lives in BigQuery as a model object
-- Predict segment for new customers:
SELECT * FROM ML.PREDICT(
    MODEL `fmn-sandbox.analytics.kmeans_customer_segments`,
    (SELECT * FROM `fmn-sandbox.analytics.int_rfm_features` WHERE UNIQUE_ID = 'some_id')
)

5 clusters, Davies-Bouldin score 1.295. Input: 9 normalised RFM features. Output: centroid_id mapped to segment names via the naming logic in sql/04_marts/mart_cluster_output.sql.

Churn prediction (logistic regression)

-- Evaluate model performance
SELECT * FROM ML.EVALUATE(
    MODEL `fmn-sandbox.analytics.churn_logistic_model`
)

83.1% accuracy, F1 0.556. Input: 15 features including recency, frequency, spend trends, merchant diversity change. Binary output: churned (1) or retained (0), with a probability score.

CLV prediction (linear regression)

Predicts 6-month future spend from 12-month historical features. Output: predicted_clv, tiered into Platinum through Basic.

Building custom audiences

The CTE pattern. Define who's in the audience, then profile them.

-- Template: any audience you want to build
WITH my_audience AS (
    SELECT DISTINCT cs.UNIQUE_ID
    FROM `fmn-sandbox.analytics.int_customer_category_spend` cs
    JOIN `fmn-sandbox.staging.stg_customers` c ON cs.UNIQUE_ID = c.UNIQUE_ID
    WHERE cs.DESTINATION = 'CLICKS'       -- change store
        AND c.gender_label = 'Female'     -- change demographics
        AND c.age BETWEEN 18 AND 40       -- change demographics
)
-- Profile: size + demographics
SELECT
    COUNT(*) AS audience_size,
    ROUND(AVG(c.age), 1) AS avg_age,
    ROUND(AVG(c.estimated_income), 0) AS avg_income,
    ROUND(COUNTIF(c.gender_label = 'Female') * 100.0
        / NULLIF(COUNT(*), 0), 1) AS pct_female
FROM my_audience a
JOIN `fmn-sandbox.staging.stg_customers` c ON a.UNIQUE_ID = c.UNIQUE_ID
-- Enrich: what segments are they in?
WITH my_audience AS (
    -- same CTE as above
    SELECT DISTINCT cs.UNIQUE_ID
    FROM `fmn-sandbox.analytics.int_customer_category_spend` cs
    WHERE cs.DESTINATION = 'CLICKS'
)
SELECT co.segment_name, COUNT(*) AS n
FROM my_audience a
JOIN `fmn-sandbox.marts.mart_cluster_output` co ON a.UNIQUE_ID = co.UNIQUE_ID
GROUP BY 1
ORDER BY n DESC
-- Enrich: what else do they buy?
WITH my_audience AS (
    SELECT DISTINCT cs.UNIQUE_ID
    FROM `fmn-sandbox.analytics.int_customer_category_spend` cs
    WHERE cs.DESTINATION = 'CLICKS'
)
SELECT cs2.CATEGORY_TWO, COUNT(DISTINCT cs2.UNIQUE_ID) AS customers,
    ROUND(SUM(cs2.dest_spend), 0) AS spend
FROM my_audience a
JOIN `fmn-sandbox.analytics.int_customer_category_spend` cs2
    ON a.UNIQUE_ID = cs2.UNIQUE_ID
WHERE cs2.DESTINATION != 'CLICKS'
GROUP BY 1
ORDER BY spend DESC
LIMIT 10

Python / notebook patterns

All examples use google-cloud-bigquery with pandas. Works in Colab, Jupyter, or any notebook.

from google.cloud import bigquery
import pandas as pd

PROJECT = 'fmn-sandbox'
client = bigquery.Client(project=PROJECT, location='africa-south1')

def bq(sql):
    """Run a query and return a DataFrame."""
    return client.query(sql).to_dataframe()

Audience builder function

def build_audience(destination, gender=None, age_min=None, age_max=None,
                   income_groups=None, exclude_income=None):
    """Build a proxy audience from transaction + demographic data.

    Returns DataFrame with audience_size, avg_age, avg_income, pct_female.
    """
    filters = [f"cs.DESTINATION = '{destination}'"]
    if gender:
        filters.append(f"c.gender_label = '{gender}'")
    if age_min is not None and age_max is not None:
        filters.append(f"c.age BETWEEN {age_min} AND {age_max}")
    if income_groups:
        quoted = ', '.join(f"'{g}'" for g in income_groups)
        filters.append(f"c.income_group IN ({quoted})")
    if exclude_income:
        quoted = ', '.join(f"'{g}'" for g in exclude_income)
        filters.append(f"c.income_group NOT IN ({quoted})")

    where = ' AND '.join(filters)

    return bq(f"""
        SELECT
            COUNT(DISTINCT cs.UNIQUE_ID) AS audience_size,
            ROUND(AVG(c.age), 1) AS avg_age,
            ROUND(AVG(c.estimated_income), 0) AS avg_income,
            ROUND(COUNTIF(c.gender_label = 'Female') * 100.0
                / NULLIF(COUNT(*), 0), 1) AS pct_female,
            ROUND(SUM(cs.dest_spend), 0) AS total_spend,
            ROUND(AVG(cs.dest_spend), 0) AS avg_spend
        FROM `fmn-sandbox.analytics.int_customer_category_spend` cs
        JOIN `fmn-sandbox.staging.stg_customers` c
            ON cs.UNIQUE_ID = c.UNIQUE_ID
        WHERE {where}
    """)

# Usage
revlon = build_audience('CLICKS', gender='Female', age_min=18, age_max=40,
                        income_groups=['R5.5k-R13.5k', 'R13.5k-R23.5k', 'R23.5k-R32.5k'])
print(revlon)

eucerin = build_audience('CLICKS', gender='Female', age_min=25, age_max=55,
                         income_groups=['R32.5k-R56k', 'R56k+'])
print(eucerin)

vital = build_audience('CLICKS', age_min=30, age_max=60,
                       exclude_income=['R0-R5.5k', 'Unknown'])
print(vital)

# WRONG — this crashes in BigQuery:
# SELECT MEDIAN(total_spend) FROM ...

# RIGHT — use APPROX_QUANTILES:
percentiles = bq("""

        APPROX_QUANTILES(total_spend, 4)[OFFSET(3)] AS q75,

BigQuery gotchas

Things that work in PostgreSQL/MySQL but break in BigQuery.

GotchaWrongRight
No MEDIAN() SELECT MEDIAN(spend) SELECT APPROX_QUANTILES(spend, 2)[OFFSET(1)]
ROWS is a keyword SELECT rows FROM ... SELECT `rows` FROM ... or rename the column
No ILIKE WHERE name ILIKE '%click%' WHERE UPPER(name) LIKE '%CLICK%'
No LIMIT in subqueries (some contexts) FROM (SELECT ... LIMIT 10) Use a CTE instead: WITH top10 AS (SELECT ... LIMIT 10)
Date arithmetic date - 30 DATE_SUB(date, INTERVAL 30 DAY)
Division by zero a / b a / NULLIF(b, 0) or SAFE_DIVIDE(a, b)
String aggregation GROUP_CONCAT(x) STRING_AGG(x, ', ')
Current date NOW() CURRENT_DATE() or CURRENT_TIMESTAMP()
Type casting CAST(x AS VARCHAR) CAST(x AS STRING)
Top N per group LIMIT inside GROUP BY ROW_NUMBER() OVER (PARTITION BY ... ORDER BY ...) = 1
Approximate distinct count COUNT(DISTINCT x) on huge data APPROX_COUNT_DISTINCT(x) — faster, ~1% error, much cheaper
Cost control: Always check estimated bytes before running. In the BQ console, click the green checkmark (dry run) to see "This query will process X GB" before executing. On stg_transactions, add a date filter to avoid scanning the full 2.2B rows.
