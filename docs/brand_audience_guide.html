Building brand proxy audiences from transaction data

A complete guide with SQL, reasoning, and exercises. Written for someone who knows basic SQL and wants to understand how we built the Clicks brand audiences (Revlon, Eucerin, Vital, Cantu).

The problem we're solving

A client (Webfluential) wants us to build targetable audiences for brands sold at Clicks:

BrandProduct typeTarget customer
RevlonMass-market makeupYoung women, mid-income
Eucerin / La Roche-PosayPremium dermocosmeticsWomen, higher income, 25-55
VitalVitamins & supplementsHealth-conscious, all genders, 30-60
CantuEthnic haircareWomen with textured hair, 18-45

The problem: our data doesn't have brand or product information. We see that someone spent R350 at Clicks on 15 March. We do NOT see that they bought Revlon lipstick and Eucerin moisturiser.

What we have vs what we don't

We have: Which store they shopped at (DESTINATION), what category that store is in (CATEGORY_TWO), how much they spent, when they went, and their demographics (age, gender, estimated income).

We don't have: What specific products or brands they bought. No SKU data, no barcode data, no basket-level detail.

We use a technique called demographic proxy segmentation. We can't identify Revlon buyers directly, so we identify the TYPE of person most likely to buy Revlon — then find all Clicks shoppers who match that type. The store is the starting point. Demographics are the filter.

Understanding our data

Three tables matter for this exercise:

1. int_customer_category_spend

One row per customer × destination. Shows how much each person spent at each store. This is our main table — it tells us WHO shops WHERE.

Key columns: UNIQUE_ID (customer), DESTINATION (store name), CATEGORY_TWO (store category), dest_spend (how much they spent there)

2. stg_customers

One row per customer. Demographics. This tells us WHO each person is.

Key columns: UNIQUE_ID, age, gender_label (Male/Female), estimated_income, age_group (18-25, 26-35, etc.), income_group (R0-R5.5k, R5.5k-R13.5k, etc.)

3. mart_cluster_output

One row per customer. Their ML-assigned behavioural segment (Champions, Loyal, Mid-Tier, At Risk, Dormant). This tells us HOW each person behaves.

Key columns: UNIQUE_ID, segment_name

Always check what values actually exist in the data before writing filters. Never assume. Run discovery queries first.

The approach (overview)

Five steps, each building on the previous one:

StepWhatWhy
1. DiscoveryFind the exact store name, categories, demographic valuesSo our filters match actual data, not what we assume
2. Base audienceCount everyone who shops at ClicksThis is our universe — every possible buyer
3. Brand filtersFilter by demographics matching each brandNarrow to the most likely buyers for each brand
4. EnrichmentAdd behavioural data, co-shopping patternsTurn a simple filter into a rich audience profile
5. OutputFormat as HTML reportShareable deliverable for the client

Step 1: Discovery queries

Before writing a single audience query, you MUST know exactly what's in the data. Here's what we ran and why:

1aFind the exact store name

SELECT DESTINATION, CATEGORY_TWO, customers,
    ROUND(total_spend, 0) AS spend, spend_rank
FROM `fmn-sandbox.marts.mart_destination_benchmarks`
WHERE UPPER(DESTINATION) LIKE '%CLICK%'
ORDER BY spend DESC LIMIT 10
Store names in the data might not match what you expect. It could be "CLICKS", "Clicks", "CLICKS PHARMACY", or something else. This query finds the exact string so your filters work. We found it's called CLICKS in category Pharmacies and Wellbeing.

1bFind health/beauty categories

SELECT DISTINCT CATEGORY_TWO
FROM `fmn-sandbox.marts.mart_destination_benchmarks`
WHERE UPPER(CATEGORY_TWO) LIKE '%HEALTH%'
    OR UPPER(CATEGORY_TWO) LIKE '%PHARM%'
    OR UPPER(CATEGORY_TWO) LIKE '%BEAUTY%'
    OR UPPER(CATEGORY_TWO) LIKE '%CARE%'
ORDER BY 1
We need to know what related categories exist. Found: Childcare/Daycare, Gap Cover & Health Insurance, Health and Beauty, Healthcare Professionals, Nail and Beauty Salon, Pet Health Insurance, Pharmacies and Wellbeing. This tells us Clicks lives in "Pharmacies and Wellbeing" and there are other health/beauty categories we could use for enrichment later.

1cFind demographic values

-- What gender values exist?
SELECT gender_label, COUNT(*) AS n
FROM `fmn-sandbox.staging.stg_customers`
GROUP BY 1 ORDER BY 2 DESC

-- What income groups exist?
SELECT income_group, COUNT(*) AS n
FROM `fmn-sandbox.staging.stg_customers`
GROUP BY 1 ORDER BY 1

-- What age groups exist?
SELECT age_group, COUNT(*) AS n
FROM `fmn-sandbox.staging.stg_customers`
GROUP BY 1 ORDER BY 1
You MUST use the exact values that exist in the data. If you filter for income_group = 'R10k-R20k' but the actual value is 'R13.5k-R23.5k', your query returns zero rows and you'll think nobody matches. Always discover first.

What we found:

FieldValues

income_groupR0-R5.5k, R5.5k-R13.5k, R13.5k-R23.5k, R23.5k-R32.5k, R32.5k-R56k, R56k+, Unknown

1dCount the Clicks base

SELECT COUNT(DISTINCT UNIQUE_ID) AS clicks_shoppers
FROM `fmn-sandbox.analytics.int_customer_category_spend`

Result: 2,554,110 people shop at Clicks. This is our universe.

Run all of these yourself in the BigQuery console. Change 'CLICKS' to 'DIS-CHEM' or 'PICK N PAY' and see what you get. Understanding the data shape is the most important skill.

Step 2: Base audience profile

Before filtering for brands, understand the full Clicks shopper base:

SELECT
    COUNT(DISTINCT cs.UNIQUE_ID) AS total_shoppers,
    ROUND(SUM(cs.dest_spend), 0) AS total_spend,
    ROUND(AVG(cs.dest_spend), 0) AS avg_spend,
    ROUND(AVG(c.age), 1) AS avg_age,
    ROUND(AVG(c.estimated_income), 0) AS avg_income,
    ROUND(
        COUNTIF(c.gender_label = 'Female') * 100.0
        / NULLIF(COUNT(*), 0),
    1) AS pct_female
FROM `fmn-sandbox.analytics.int_customer_category_spend` cs
JOIN `fmn-sandbox.staging.stg_customers` c
    ON cs.UNIQUE_ID = c.UNIQUE_ID
WHERE cs.DESTINATION = 'CLICKS'

What this query does, line by line

FROM int_customer_category_spend cs — start with the spend table (who shops where)

JOIN stg_customers c ON cs.UNIQUE_ID = c.UNIQUE_ID — join demographics so we can see age/gender/income

WHERE cs.DESTINATION = 'CLICKS' — filter to only Clicks shoppers

COUNT(DISTINCT cs.UNIQUE_ID) — count unique people (not transactions)

AVG(cs.dest_spend) — average spend per person at Clicks

COUNTIF(c.gender_label = 'Female') * 100.0 / NULLIF(COUNT(*), 0) — percentage that are female. The NULLIF prevents division by zero.

This gives you the baseline. If the average Clicks shopper is 40 years old and earns R17k, you know what "normal" looks like. Brand audiences will deviate from this baseline — Revlon skews younger and female, Vital skews older and mixed gender.

Step 3: Brand-specific demographic filters

This is the core technique. We take the same query from Step 2 and add demographic filters that match each brand's target market.

3aRevlon — mass-market makeup

Logic: Revlon is affordable, widely available makeup. The typical buyer is female, younger (18-40), with mid-range income. Not ultra-premium, not budget-only.

SELECT
    COUNT(DISTINCT cs.UNIQUE_ID) AS audience_size,
    ROUND(AVG(c.age), 1) AS avg_age,
    ROUND(AVG(c.estimated_income), 0) AS avg_income,
    ROUND(
        COUNTIF(c.gender_label = 'Female') * 100.0
        / NULLIF(COUNT(*), 0),
    1) AS pct_female,
    ROUND(SUM(cs.dest_spend), 0) AS total_spend,
    ROUND(AVG(cs.dest_spend), 0) AS avg_spend
FROM `fmn-sandbox.analytics.int_customer_category_spend` cs
JOIN `fmn-sandbox.staging.stg_customers` c
    ON cs.UNIQUE_ID = c.UNIQUE_ID
WHERE cs.DESTINATION = 'CLICKS'
    AND c.gender_label = 'Female'
    AND c.income_group IN (
        'R5.5k-R13.5k',
        'R13.5k-R23.5k',
        'R23.5k-R32.5k'
    )
    AND c.age BETWEEN 18 AND 40

Why these specific filters?

gender_label = 'Female' — Revlon is primarily a women's makeup brand

income_group IN ('R5.5k-R13.5k', 'R13.5k-R23.5k', 'R23.5k-R32.5k') — mid-income. We excluded R0-R5.5k (likely not regular cosmetics buyers) and R32.5k+ (more likely to buy premium brands like MAC or Clinique). These income groups come from discovery query 1c.

age BETWEEN 18 AND 40 — Revlon's core demographic. Under 18 is excluded from our data anyway. Over 40 are more likely premium skincare buyers.

3bEucerin / La Roche-Posay — premium dermocosmetics

Logic: These are doctor-recommended, expensive skincare products. The buyer is female, older, higher income.

WHERE cs.DESTINATION = 'CLICKS'
    AND c.gender_label = 'Female'
    AND c.income_group IN ('R32.5k-R56k', 'R56k+')
    AND c.age BETWEEN 25 AND 55
Higher income brackets because these products cost R200-R500 per item. Age 25-55 because skincare concerns (acne scarring, ageing, pigmentation) peak in this range. Still female-only because the brand primarily targets women.

3cVital — vitamins & supplements

Logic: Vitamins are bought by both genders. Health-conscious, established adults with disposable income for supplements.

WHERE cs.DESTINATION = 'CLICKS'
    AND c.income_group NOT IN ('R0-R5.5k', 'Unknown')
    AND c.age BETWEEN 30 AND 60
No gender filter — men and women buy vitamins equally. We excluded the lowest income bracket (supplements are discretionary spending) and Unknown. Age 30-60 because younger people rarely buy vitamins regularly, and 60+ may already be on prescription medication.

3dCantu — ethnic haircare

Logic: Cantu is affordable ethnic/textured haircare. The buyer is female, broad age range, all income levels.

WHERE cs.DESTINATION = 'CLICKS'
    AND c.gender_label = 'Female'
    AND c.age BETWEEN 18 AND 45
Female because Cantu primarily targets women's hair. No income filter because Cantu is affordable (R80-R150 per product) and used across all income levels. Age 18-45 covers students through young professionals who are active in natural hair care.
These filters are informed opinions, not scientific truth. A 50-year-old man could buy Revlon for his daughter. The filters define the MOST LIKELY buyers, not the only buyers. That's why it's called a proxy audience — it's an approximation.

Step 4: Enrichment queries

A demographic filter alone gives you a number ("480K people"). Enrichment turns it into a profile ("480K young women who also shop at Woolworths, are in the Mid-Tier segment, and 65% are in Gauteng"). This is what makes the audience useful for campaign planning.

4aAge breakdown

SELECT c.age_group, COUNT(DISTINCT cs.UNIQUE_ID) AS n
FROM `fmn-sandbox.analytics.int_customer_category_spend` cs
JOIN `fmn-sandbox.staging.stg_customers` c
    ON cs.UNIQUE_ID = c.UNIQUE_ID
WHERE cs.DESTINATION = 'CLICKS'
    AND c.gender_label = 'Female'
    AND c.income_group IN (
        'R5.5k-R13.5k', 'R13.5k-R23.5k', 'R23.5k-R32.5k'
    )
    AND c.age BETWEEN 18 AND 40
    AND c.age_group IS NOT NULL
    AND c.age_group != 'Unknown'
GROUP BY 1
ORDER BY 1
Shows the age distribution WITHIN the audience. Even though we filtered 18-40, we want to see: are they mostly 18-25 or mostly 36-45? This matters for creative — you'd use different ad imagery for a 20-year-old vs a 38-year-old.

4bIncome breakdown

-- Same structure, swap age_group for income_group
SELECT c.income_group, COUNT(DISTINCT cs.UNIQUE_ID) AS n
FROM `fmn-sandbox.analytics.int_customer_category_spend` cs
JOIN `fmn-sandbox.staging.stg_customers` c
    ON cs.UNIQUE_ID = c.UNIQUE_ID
WHERE cs.DESTINATION = 'CLICKS'
    AND c.gender_label = 'Female'
    AND c.income_group IN (
        'R5.5k-R13.5k', 'R13.5k-R23.5k', 'R23.5k-R32.5k'
    )
    AND c.age BETWEEN 18 AND 40
    AND c.income_group IS NOT NULL
    AND c.income_group != 'Unknown'
GROUP BY 1
ORDER BY 1

4cTop co-shopped categories (where else they spend)

This is a two-step query using a CTE (Common Table Expression):

WITH audience AS (
    -- Step 1: Find all UNIQUE_IDs that match our brand filter
    SELECT DISTINCT cs.UNIQUE_ID
    FROM `fmn-sandbox.analytics.int_customer_category_spend` cs
    JOIN `fmn-sandbox.staging.stg_customers` c
        ON cs.UNIQUE_ID = c.UNIQUE_ID
    WHERE cs.DESTINATION = 'CLICKS'
        AND c.gender_label = 'Female'
        AND c.income_group IN (
            'R5.5k-R13.5k', 'R13.5k-R23.5k', 'R23.5k-R32.5k'
        )
        AND c.age BETWEEN 18 AND 40
)
-- Step 2: For those people, what ELSE do they buy?
SELECT
    cs2.CATEGORY_TWO AS category,
    COUNT(DISTINCT cs2.UNIQUE_ID) AS customers,
    ROUND(SUM(cs2.dest_spend), 0) AS spend
FROM audience a
JOIN `fmn-sandbox.analytics.int_customer_category_spend` cs2
    ON a.UNIQUE_ID = cs2.UNIQUE_ID
WHERE cs2.DESTINATION != 'CLICKS'  -- exclude Clicks itself
GROUP BY 1
ORDER BY spend DESC
LIMIT 8

How the CTE works

WITH audience AS (...) — this creates a temporary table called "audience" containing just the UNIQUE_IDs that match our Revlon filter. Think of it as a saved list of people.

The main query then joins this list back to int_customer_category_spend to find what OTHER stores/categories these same people shop at. The WHERE cs2.DESTINATION != 'CLICKS' excludes Clicks itself — we already know they shop there.

This tells the advertiser: "Your Revlon audience also spends heavily on Groceries (R4.2B), Clothing (R1.8B), and Fuel (R1.1B)." This is useful for campaign planning — if they also watch content about food and fashion, that's where to place your ads.

4dBehavioural segments

WITH audience AS (
    SELECT DISTINCT cs.UNIQUE_ID
    FROM `fmn-sandbox.analytics.int_customer_category_spend` cs
    JOIN `fmn-sandbox.staging.stg_customers` c
        ON cs.UNIQUE_ID = c.UNIQUE_ID
    WHERE cs.DESTINATION = 'CLICKS'
        AND c.gender_label = 'Female'
        AND c.income_group IN (
            'R5.5k-R13.5k', 'R13.5k-R23.5k', 'R23.5k-R32.5k'
        )
        AND c.age BETWEEN 18 AND 40
)
SELECT co.segment_name, COUNT(*) AS n
FROM audience a
JOIN `fmn-sandbox.marts.mart_cluster_output` co
    ON a.UNIQUE_ID = co.UNIQUE_ID
GROUP BY 1
ORDER BY n DESC
This shows what proportion of the audience are Champions (high value), Mid-Tier (growth opportunity), or At Risk (might need retention). If 40% are Champions, it's a premium audience despite being "mid-income." If 60% are Dormant, the audience has lower engagement and you might reconsider the targeting.

Step 5: Putting it together

The Python script generate_clicks_brands.py runs all these queries automatically for each brand, collects the results, and generates an HTML report. The structure is:

for each brand:
    1. Run demographics query (size, avg age, income, gender)
    2. Run age breakdown query
    3. Run income breakdown query
    4. Run co-shopping categories query
    5. Run segment breakdown query
    6. Format results into an HTML card

The HTML card shows: audience size, key stats (avg spend, age, income, % female), activation channels (Meta/Google/TikTok), demographic breakdowns as bar charts, co-shopping categories, behavioural segments, and recommended campaign use cases.

Why we use Python instead of just SQL

SQL gives you the data. Python formats it into a presentable report. The SQL results are tables of numbers — the Python script turns them into a visual HTML page with bar charts, color-coded cards, and formatted numbers. You could do the SQL in BigQuery console and manually format — the script just automates it so you can regenerate in 30 seconds when data changes.

The full pattern

Every brand audience follows the same template. Only the WHERE clause changes:

-- Template: change the filter for each brand
SELECT COUNT(DISTINCT cs.UNIQUE_ID) AS audience_size,
    ROUND(AVG(c.age), 1) AS avg_age,
    ROUND(AVG(c.estimated_income), 0) AS avg_income,
    ROUND(
        COUNTIF(c.gender_label = 'Female') * 100.0
        / NULLIF(COUNT(*), 0),
    1) AS pct_female,
    ROUND(SUM(cs.dest_spend), 0) AS total_spend,
    ROUND(AVG(cs.dest_spend), 0) AS avg_spend
FROM `fmn-sandbox.analytics.int_customer_category_spend` cs
JOIN `fmn-sandbox.staging.stg_customers` c
    ON cs.UNIQUE_ID = c.UNIQUE_ID
WHERE cs.DESTINATION = '[STORE_NAME]'
    AND [BRAND_SPECIFIC_FILTERS]

The enrichment queries all follow the same CTE pattern — define the audience, then join to another table to see what else they do.

Key SQL techniques used

TechniqueWhat it doesExample
COUNT(DISTINCT x)Counts unique values, not duplicatesA person shopping 10 times counts as 1
COUNTIF(condition)Counts rows matching a conditionCOUNTIF(gender='Female') counts women
NULLIF(x, 0)Prevents division by zero100.0 / NULLIF(COUNT(*), 0)
WITH ... ASCreates a temporary named resultCTE to define audience, then reuse it
ROUND(x, n)Rounds to n decimal placesROUND(AVG(age), 1) gives 34.2
IN ('a', 'b')Matches any value in a listBetter than multiple OR conditions
BETWEEN x AND yRange filter inclusiveage BETWEEN 18 AND 40
NOT IN ('a', 'b')Excludes specific valuesUsed for Vital to exclude lowest income

Exercises — practice this yourself

Exercise 1: Dis-Chem base audience. Run the discovery queries but for Dis-Chem instead of Clicks. What's the exact destination name? How many shoppers? What's the gender split? Compare to Clicks.
Exercise 2: Build a Nike audience. Nike is sold at sports/clothing stores. Find the right destination name, pick demographic filters for a Nike buyer (hint: consider gender, age, and what kind of income group buys premium sportswear). Run the audience size query.
Exercise 3: Modify the Cantu filter. The current Cantu audience has no income filter. What happens to the audience size if you exclude the lowest income bracket (R0-R5.5k)? Does the average spend per customer go up? Should we exclude them?
Exercise 4: Co-shopping overlap. Using the CTE pattern from Step 4c, find out: what percentage of Revlon audience members also appear in the Cantu audience? (Hint: create two CTEs, one for each audience, then count the intersection.)
Exercise 5: Build a Woolworths Food audience. Woolworths Food is premium grocery. Define the target demographic (hint: think about who buys organic, premium groceries). Write the filters. Run it. How does the audience size compare to Clicks?
Exercise 6: Seasonal twist. Using stg_transactions, find Clicks shoppers who spend 1.5x their monthly average in December. These are "Festive Gift Buyers" — a seasonal audience for holiday campaigns. (Warning: this query scans the big transaction table. Add a date filter to only look at the last 14 months.)
For exercises 4-6, check with Prosper before running expensive queries. Anything that touches stg_transactions scans billions of rows and costs money. Queries on int_customer_category_spend and stg_customers are cheap.
