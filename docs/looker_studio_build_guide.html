Looker Studio Build Guide

Step-by-step instructions for building the NAV Analytics dashboard in Looker Studio. Follow each step exactly. Total time: ~3-4 hours.

Prerequisites Data sources Overview Client pitch Segments Churn & CLV Categories Audiences Anonymization Polish

Prerequisites

Run the new SQL files

Before building the dashboard, you need the new mart tables. On the work machine:

cd ~/fnb-nav-pipeline
git pull

# Run the new marts (step 4 only — doesn't re-run staging or ML)
bash scripts/run.sh sandbox 4

# Run the new views
bq query --use_legacy_sql=false --project_id=fmn-sandbox < sql/05_looker_views/v_pitch_views.sql
bq query --use_legacy_sql=false --project_id=fmn-sandbox < sql/05_looker_views/v_dashboard_views.sql

Verify everything exists:

bq ls fmn-sandbox:marts | grep -E "mart_store_loyalty|mart_store_time|mart_audience"

Open Looker Studio

Go to lookerstudio.google.com. Sign in with your FNB Google account. Click + Create → Report. Name it NAV Analytics Dashboard.

Connect data sources

You need to connect 8 data sources. Each one is a BigQuery table or view. You do this once and all pages share them.

Create the page

The first page of your report is already created. Right-click the page tab at the top and rename it to Overview.

Add 5 scorecard KPIs across the top

For each KPI: Add a chart → Scorecard. Data source: DS_Overview.

Arrange them in a row across the top. Set font size to 24px for the number, 10px for the label.

Add segment donut chart

Add a chart → Pie chart (donut). Data source: DS_Segments.

  • Dimension: segment_name
  • Metric: customers (SUM)
  • Style → Donut hole: 60%
  • Colors: Champions=#0f172a, Loyal=#1e3a5f, Mid-Tier=#4CAF50, At Risk=#FF9800, Dormant=#f44336

Add revenue concentration bar chart

Add a chart → Bar chart (grouped). Data source: DS_Segments.

  • Dimension: segment_name
  • Metrics: Create two calculated fields:
    • pct_customers: already exists
    • pct_revenue: already exists (from mart_cluster_profiles)
  • Style: grouped bars, first series = grey (#94a3b8), second = navy (#0f172a)
If pct_customers and pct_revenue aren't in the view, add them as calculated fields: customers / SUM(customers) formatted as percent.

Page 2 Client Pitch

This is the most complex page. It has filters, KPIs, competitor charts, loyalty, time patterns, trends, demographics, and geography — all on one page.

Create page and add filters

Click the + tab to add a new page. Rename to Client Pitch.

Add 2 dropdown filter controls at the top:

  • Add a control → Drop-down list. Data source: DS_ClientPitch. Control field: CATEGORY_TWO. Label: "Category".
  • Add a control → Drop-down list. Data source: DS_ClientPitch. Control field: DESTINATION. Label: "Client".
Both filters must use the SAME data source (DS_ClientPitch) so they cross-filter. When you select a category, the client dropdown should only show destinations in that category.

Add 8 KPI scorecards

Data source: DS_ClientPitch. All are SUM or AVG aggregations.

Arrange in 2 rows of 4. Format: large number, small label below.

Add competitor benchmark chart

Add a chart → Bar chart (horizontal). Data source: DS_ClientPitch.

  • Dimension: DESTINATION
  • Metric: market_share_pct (AVG)
  • Sort: market_share_pct descending
  • Style: horizontal bars, single color #0f172a

Duplicate this chart. Change metric to spend_per_customer for the second chart.

To highlight the selected client in a different color, use a conditional formatting rule or create a calculated field: IF(DESTINATION = "WOOLWORTHS", "Client", "Competitor") and use it as the color dimension.

Add store loyalty chart

Add a chart → Bar chart. Data source: DS_StoreLoyalty.

  • Dimension: DESTINATION
  • Metric: pct_loyal_50 (AVG) — "% of customers with 50%+ share of wallet"
  • Sort: pct_loyal_50 descending
  • Add filter: same CATEGORY_TWO filter from the page dropdown

Add shopping time table

Add a chart → Table. Data source: DS_StoreTime.

  • Dimension: DESTINATION
  • Metrics: pct_early_morning, pct_mid_morning, pct_afternoon, pct_evening, pct_weekend
  • Add filter: same CATEGORY_TWO filter
  • Style: heatmap on columns (Style → Column → Apply heatmap)

Add monthly trend chart

Add a chart → Time series. Data source: DS_Trends.

  • Date dimension: month
  • Breakdown dimension: DESTINATION (this gives one line per store)
  • Metric: total_spend (SUM)
  • Add filter: same CATEGORY_TWO filter
To show just the client + category total: create a calculated field IF(DESTINATION = selected_client, DESTINATION, "Category Total") and use as breakdown.

Add demographics charts

Three charts in a row. Data source: DS_Demographics. All filtered by CATEGORY_TWO.

Pie chart (donut)gender_labelcustomers (SUM)

Add geography chart

Add a chart → Bar chart (horizontal). Data source: DS_Geo.

  • Dimension: PROVINCE
  • Metric: total_spend (SUM)
  • Sort: total_spend descending
  • Filter: same CATEGORY_TWO filter
For drill-down: In the chart settings, enable "Drill down" and add MUNICIPALITY as the second dimension. Users can click a province to see municipalities inside it.

Add day-of-week chart

Add a chart → Bar chart. Data source: DS_StoreTime.

  • Dimension: Create a calculated field that maps peak_day_number to day names
  • Or simply use a table with columns: pct_sunday through pct_saturday
  • Filter: same CATEGORY_TWO and DESTINATION filters

Anonymization (Internal vs External)

Duplicate the Client Pitch page

Right-click the Client Pitch page tab → Duplicate page. Rename the copy to Client Pitch (External).

Switch data source on the external page

On the external page, select ALL charts that show destination names (competitor benchmarks, time patterns, loyalty). For each chart:

  1. Click the chart
  2. In the right panel, click the data source dropdown
  3. Change from DS_PitchInternal to DS_PitchExternal
  4. Change the dimension from DESTINATION to display_name

Now the external page shows "Competitor #1", "Competitor #2" etc. while the internal page shows real names. The selected client always shows their real name on both pages.

Keep both pages in the report. When presenting internally, navigate to "Client Pitch". When presenting to a client, navigate to "Client Pitch (External)". You can also hide pages: right-click tab → "Hide in view mode".

Polish & finishing touches

Set up drill-downs

For any chart where you want drill-down, click the chart → Data panel → turn on Drill down. Then add dimensions in order:

ChartDrill path
Geography bar chartPROVINCE → MUNICIPALITY
Monthly trendmonth → week (if available)
Age chartage_group → income_group
Churn risk tablechurn_risk_level → reason_1
Category scorecardCATEGORY_TWO → DESTINATION

Add download buttons

Looker Studio has built-in export. Users can:

  • Click ⋮ (three dots) on any chart → Export data → CSV or Google Sheets
  • Click File → Download → PDF (exports entire page)

No extra work needed. Just make sure tables have clean column names.

Apply theme / branding

Theme and layout (top menu) → Custom:

  • Primary color: #1e3a5f
  • Secondary color: #2E75B6
  • Font: Google Sans or Roboto
  • Background: #fafbfc
  • Chart background: #ffffff

Add header

On each page, add a text box at the top with:

  • Title: "NAV Analytics Dashboard"
  • Subtitle: page name
  • Bottom-right: "fmn-sandbox · Internal"

Set default filters

For the Client Pitch page, set default filter values so it loads with a useful view:

  • Category: "Groceries" (or whatever your most-used category is)
  • Client: first destination by spend in that category

Click the filter control → Properties → Default value.

Test everything

  • Switch to View mode (eye icon top right)
  • Change category dropdown — do all charts update?
  • Change client dropdown — do KPIs update?
  • Click a province in the geo chart — does it drill to municipality?
  • Go to Client Pitch (External) — are competitors anonymized?
  • Export a table as CSV — does it download?
  • Download page as PDF — does it look clean?
  • Check every page has data (no empty charts)
  • Check Audiences page — do type filters work?

Share the dashboard

Click Share (top right) → Add emails or set to "Anyone in the organization with the link can view".

For external sharing: File → Download as PDF and send the file. Never share the live dashboard link externally.
